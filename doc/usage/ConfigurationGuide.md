# 系统配置与使用指南

当您通过任何一种方式（Docker、Cloudflare 或源码）将 GT AI Gateway 部署成功后，本文档将指导您完成从零开始的初始配置，直到能够通过网关成功调用大模型 API。

---

## 第一步：登录系统后台

1. 打开浏览器，访问您部署成功后的地址（例如 `http://localhost:8787` 或分配给您的 Cloudflare Workers 域名）。
2. 页面会提示您输入密码，请输入您在部署时设置的管理员密钥（即环境变量 `ROOT_TOKEN` 的值）。
3. 登录成功后，您将进入系统的主控制台面板。

---

## 第二步：配置供应商 (Vendors)

网关本身不生产 AI 能力，您需要把上游模型厂商（例如 OpenAI、Anthropic、智谱等）的 API Key 配置进来。

1. 在左侧菜单栏点击 **“渠道管理” (Vendors)**。
2. 点击右上角的 **“添加渠道”** 按钮。
3. 填写必要的信息：
   - **名称**：例如 `官方 OpenAI`。
   - **Endpoint**：输入供应商的接口地址。比如 `https://api.openai.com`。
   - **API Key**：填入您在对应厂商平台申请到的 Secret Key。
4. 保存后，您的系统中就成功接入了一个上游供应商。

---

## 第三步：配置模型路由 (Models)

接入供应商后，您需要告诉网关：当用户请求某个特定模型时，应该转发给哪个供应商处理。

1. 在左侧菜单栏点击 **“模型管理” (Models)**。
2. 点击右上角的 **“添加模型”** 按钮。
3. 填写模型映射关系：
   - **名称 (Model ID)**：这是对用户暴露的模型名称。比如填入 `gpt-4o`。
   - **映射目标**：在弹出的列表中，勾选您刚才创建的渠道（如 `官方 OpenAI`）。
   - *高级玩法*：您可以将同一个 `gpt-4o` 映射给多个渠道（比如官方和一个代理商），网关将根据您设定的权重和优先级，自动进行负载均衡和故障转移。
4. 保存配置。

---

## 第四步：创建用户与令牌 (Users)

现在您需要为调用方（可以是您自己，也可以是您的客户）分配调用凭证。

1. 在左侧菜单栏点击 **“用户管理” (Users)**。
2. 点击右上角的 **“添加用户”** 按钮。
3. 填写用户信息：
   - 输入用户的名称。
   - 设定该用户的**额度**。系统支持基于美元价值的精准计费，您可以给用户充值比如 10 刀的额度。
4. 保存后，点击用户列表中的 **“Token”** 或 **“查看”** 按钮，您将获取到该用户专属的 `sk-xxxxxx` 格式密钥。

---

## 第五步：开始调用网关 API

大功告成！现在，您的网关已经可以对外提供标准的 API 服务了。

请在系统左侧菜单栏点击 **“接入配置”**，该页面会自动根据您的当前环境，为您直接生成：
- OpenAI 兼容的接入端点和 Curl 测试代码示例
- Anthropic 兼容的接入端点和 Curl 测试代码示例

您只需按照“接入配置”页面上的示例代码，替换为您在第四步中获取到的用户令牌 (Token) 以及第三步中配置的模型名称，即可轻松在您的代码中发起测试。此外，您也可以将页面上提供的接入地址填写到任何第三方客户端（如 Chatbox、NextChat、Cursor 等）中无缝使用！

一旦请求成功返回内容，在网关后台的 **“请求记录”** 页面中，您就可以看到此次请求的消耗详情、耗时、关联的用户以及走的是哪个渠道了。

恭喜，您已成功配置并掌握了 GT AI Gateway！
