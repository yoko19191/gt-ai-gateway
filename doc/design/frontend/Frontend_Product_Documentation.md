# Serverless AI Gateway - 前端界面产品文档

## 一、产品概述

Serverless AI Gateway 是一个轻量级的 AI 服务网关，用于统一管理和转发多个 AI 供应商的 API 请求。前端管理界面为管理员提供可视化的操作面板，用于管理用户、供应商、模型，以及监控请求记录。

## 二、用户角色

### 2.1 管理员 (Admin)
- 拥有完整的系统访问权限
- 可以管理所有资源和配置
- 可以查看所有请求记录

### 2.2 普通用户 (Normal)
- 仅能访问 LLM API 端点（/v1/chat/completions、/v1/messages）
- 无法访问管理后台

---

## 三、功能模块

### 3.1 登录认证

#### 功能描述
管理员通过 Token 登录系统，访问管理后台。

#### 界面元素
- **登录表单**
  - Token 输入框（密码类型）
  - 登录按钮

#### 交互流程
1. 用户输入 Token
2. 点击登录按钮
3. 系统验证 Token 有效性（调用 `/` 或 `/status.json`）
4. 验证成功后进入管理后台，失败显示错误提示

#### 达到的效果
- 安全的 Token 认证机制
- 登录状态持久化（使用 localStorage 或 cookie）
- Token 验证失败时有明确的错误提示

---

### 3.2 仪表盘 (Dashboard)

#### 功能描述
展示系统概览和关键指标，包括系统状态和近期活动。

#### 界面元素
- **系统状态卡片**
  - 运行状态（正常/异常）
  - 系统版本
  - 数据库连接状态
- **统计卡片**
  - 今日请求总数
  - 今日请求成功率
  - 活跃用户数
  - 活跃模型数
- **最近请求列表**
  - 请求时间
  - 用户名称
  - 模型名称
  - 状态（成功/失败）
  - 响应时间

#### 交互流程
1. 进入系统后默认展示仪表盘
2. 数据自动刷新（可配置刷新间隔，默认 30 秒）
3. 点击"查看更多"跳转到请求记录页面

#### 达到的效果
- 一目了然的系统运行状况
- 实时更新的统计数据
- 快速了解近期请求活动

---

### 3.3 用户管理

#### 功能描述
管理系统用户，包括创建、查看、编辑用户。

#### 界面元素
- **用户列表页面**
  - 表格展示：ID、用户名、Token（部分隐藏）、类型、创建时间、操作
  - 搜索框（按用户名搜索）
  - 筛选器（按用户类型筛选）
  - "新建用户"按钮
  - 分页器

- **新建/编辑用户对话框**
  - 用户名输入框
  - Token 输入框（密码类型）
  - 类型选择器（normal/admin）
  - 保存/取消按钮

- **查看用户详情**
  - 用户名
  - Token（可复制完整 Token）
  - 类型
  - 创建时间、更新时间

#### 交互流程
1. 用户列表页展示所有用户
2. 点击"新建用户"打开创建对话框
3. 输入用户信息并保存
4. 编辑用户时预填充现有信息
5. 点击操作列的"删除"需要二次确认

#### API 调用
- `GET /user/list.json` - 获取用户列表
- `POST /user/create.json` - 创建用户
- `GET /user/:id` - 获取用户详情

#### 达到的效果
- 清晰的用户列表展示
- 方便的新建和编辑操作
- 安全的 Token 管理（默认隐藏，需要时显示完整）

---

### 3.4 供应商管理

#### 功能描述
管理 AI 服务供应商，包括阿里云、DeepSeek 等。

#### 界面元素
- **供应商列表页面**
  - 表格展示：ID、类型、名称、API 格式、URL、状态、创建时间、操作
  - 搜索框（按名称搜索）
  - 筛选器（按类型、API 格式筛选）
  - "新建供应商"按钮
  - 分页器

- **新建/编辑供应商对话框**
  - 类型选择器（aliyun/deepseek/other）
  - 名称输入框
  - Token 输入框（密码类型）
  - URL 输入框
  - API 格式选择器（openai/anthropic/google）
  - 保存/取消按钮

- **查看供应商详情**
  - 完整信息展示
  - 关联模型列表

#### 交互流程
1. 供应商列表页展示所有供应商
2. 点击"新建供应商"打开创建对话框
3. 输入供应商信息并保存
4. 编辑供应商时预填充现有信息
5. URL 输入框需要格式验证
6. 点击操作列的"删除"需要二次确认

#### API 调用
- `GET /vendor/list.json` - 获取供应商列表
- `POST /vendor/create.json` - 创建供应商
- `GET /vendor/:id` - 获取供应商详情
- `PUT /vendor/:id` - 更新供应商

#### 达到的效果
- 完整的供应商信息管理
- 表单验证确保数据有效性
- 清晰的供应商和模型关联关系

---

### 3.5 模型管理

#### 功能描述
管理各供应商下的 AI 模型。

#### 界面元素
- **模型列表页面**
  - 表格展示：ID、模型名称、供应商名称、状态、创建时间、操作
  - 搜索框（按模型名搜索）
  - 筛选器（按供应商筛选）
  - "新建模型"按钮
  - 分页器

- **新建/编辑模型对话框**
  - 供应商选择器（下拉选择已存在的供应商）
  - 模型名称输入框
  - 保存/取消按钮

#### 交互流程
1. 模型列表页展示所有模型
2. 点击"新建模型"打开创建对话框
3. 选择供应商并输入模型名称
4. 保存后返回列表
5. 编辑模型时预填充现有信息
6. 删除模型需要二次确认

#### API 调用
- `GET /model/list.json` - 获取模型列表
- `POST /model/create.json` - 创建模型
- `GET /model/:id` - 获取模型详情
- `PUT /model/:id` - 更新模型

#### 达到的效果
- 简洁的模型管理流程
- 自动关联供应商信息
- 确保模型名称的唯一性

---

### 3.6 请求记录

#### 功能描述
查看和分析所有 API 请求记录。

#### 界面元素
- **请求记录列表页面**
  - 表格展示：ID、用户、模型、请求时间、响应时间、状态、耗时
  - 搜索框（按用户名、模型名搜索）
  - 筛选器（按状态、时间范围筛选）
  - 刷新按钮
  - 分页器

- **请求详情对话框**
  - 基本信息：请求时间、用户、模型、状态、耗时
  - 请求数据（JSON 格式，可折叠）
  - 响应数据（JSON 格式，可折叠）
  - 复制数据按钮

#### 交互流程
1. 请求记录页展示历史请求（默认倒序）
2. 支持按时间范围、状态筛选
3. 点击某条记录查看详情
4. 请求数据和响应数据以 JSON 格式展示
5. 支持复制 JSON 数据

#### API 调用
- `GET /record/list.json` - 获取请求记录列表
- `GET /record/latest.json` - 获取最新记录
- `GET /record/:id` - 获取记录详情

#### 达到的效果
- 完整的请求日志追踪
- 方便的问题排查和调试
- 清晰的数据可视化

---

### 3.7 API 测试

#### 功能描述
提供 API 测试工具，方便测试聊天接口。

#### 界面元素
- **测试面板**
  - API 格式选择器（OpenAI/Anthropic）
  - 模型选择器（从已有模型中选择）
  - 消息输入区域（支持多轮对话）
  - 参数配置（temperature、max_tokens 等）
  - 发送按钮
  - 响应展示区域

#### 交互流程
1. 选择 API 格式和模型
2. 输入对话消息
3. 配置可选参数
4. 点击发送
5. 展示流式响应或完整响应
6. 支持"清除"和"重新发送"

#### 达到的效果
- 快速测试 API 功能
- 支持流式响应展示
- 直观的对话界面

## 五、设计规范

### 5.1 色彩方案

| 颜色用途 | 色值 | 说明 |
|---------|------|------|
| 主色调 | #3B82F6 (蓝色) | 主要按钮、链接、高亮 |
| 成功色 | #10B981 (绿色) | 成功状态、成功提示 |
| 警告色 | #F59E0B (橙色) | 警告提示 |
| 错误色 | #EF4444 (红色) | 错误状态、删除操作 |
| 背景色 | #F3F4F6 (浅灰) | 页面背景 |
| 卡片背景 | #FFFFFF (白色) | 卡片、表格背景 |
| 文字主色 | #1F2937 (深灰) | 主要文字 |
| 文字次色 | #6B7280 (中灰) | 次要文字 |

### 5.2 交互反馈

- **加载状态**：显示加载动画或骨架屏
- **成功操作**：显示绿色提示条（3 秒后自动消失）
- **错误提示**：显示红色错误提示
- **删除确认**：弹出确认对话框
- **表单验证**：即时显示验证错误信息

### 5.3 响应式设计

- 桌面端（> 1024px）：完整布局
- 平板端（768px - 1024px）：侧边栏可折叠
- 移动端（< 768px）：侧边栏隐藏，通过菜单按钮打开

## 六、非功能性需求

### 6.1 性能要求

- 页面首次加载时间 < 2 秒
- 列表数据加载时间 < 500ms
- 支持分页加载，避免一次性加载过多数据

### 6.2 兼容性

- 现代浏览器（Chrome、Firefox、Safari、Edge）最新版本
- 不支持 IE

### 6.3 安全性

- Token 存储使用 localStorage 时需要加密或使用 HttpOnly Cookie
- API 请求使用 HTTPS
- 敏感信息（如 Token）默认隐藏

### 6.4 可访问性

- 支持键盘导航
- 表单元素有清晰的 label
- 颜色对比度符合 WCAG 标准

## 七、技术栈

### 7.1 核心框架
- **Vue 3** + TypeScript
- **Vite** - 构建工具
- **Vue Router** - 路由管理
- **Pinia** - 状态管理

### 7.2 UI 组件库
- **Ant Design Vue** - 企业级 Vue 3 组件库（推荐）
- **Element Plus** - Vue 3 原生组件库
- **Naive UI** - 轻量级 Vue 3 组件库

### 7.3 HTTP 客户端
- **Axios** - HTTP 请求库，支持拦截器和请求取消
- **ofetch** - 轻量级 Fetch 封装

### 7.4 其他工具库
- **Day.js** - 日期时间处理
- **lodash-es** - 工具函数库
- **marked** - Markdown 渲染（用于请求数据展示）
- **highlight.js** - JSON 语法高亮
